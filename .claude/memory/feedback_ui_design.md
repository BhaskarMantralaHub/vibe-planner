When asked to make UI "rich" or "modern", don't make incremental changes — the result needs to look fundamentally different from the original in one pass.

**Why:** Multiple rounds of small tweaks (gradient backgrounds, icon swaps, spacing changes) felt like "still the same" to Bhaskar. He wants to see a dramatic transformation, not evolution.

**How to apply:**
- When redesigning UI, prototype the full vision before writing code — get alignment on specifics
- Don't propose features the user hasn't asked for (stories bar, save/share buttons were removed)
- Colored/gradient backgrounds for text posts were rejected — prefer clean, consistent card styles
- Lucide icons over react-icons/md was a good call (keep using lucide-react)
- Bottom sheets for actions were accepted — good mobile UX pattern
- Bhaskar values: clean separation, visible borders, proper interactive states, not plain HTML
