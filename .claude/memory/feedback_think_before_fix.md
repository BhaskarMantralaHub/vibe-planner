Stop pushing quick fixes that need multiple iterations. For complex issues (especially iOS Safari, mobile UX, drawer/keyboard conflicts), take time to research and plan a permanent solution BEFORE writing any code.

**Why:** Multiple failed fix attempts (password reset flow, dialog positioning, Safari textarea) frustrated the user. Each "quick fix" created a new problem.

**How to apply:**
- For iOS Safari / mobile issues: launch research agents first to understand the root cause
- Form agent councils for complex cross-platform bugs
- Present the solution plan before implementing
- Test mentally through all edge cases before coding
- One permanent fix > five iterative attempts
